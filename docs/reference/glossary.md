# Glossary

| Term | Definition |
|------|-----------|
| **Artifact** | A file stored in the `artifacts/` directory after ingestion, named using project and student metadata |
| **BRF** | Braille Ready Format — the standard file format for digital braille |
| **DAISY** | Digital Accessible Information System — an international standard for accessible e-books and digital talking books |
| **Dublin Core** | A standard set of 15 metadata elements (title, creator, subject, etc.) used to describe digital resources |
| **eBraille** | Digital braille content distributed as a file rather than embossed on paper |
| **Embosser** | A device that prints braille by pressing raised dots into paper |
| **EPUB3** | The current version of the EPUB e-book standard, with native accessibility support |
| **FTS5** | SQLite's built-in Full-Text Search version 5 extension |
| **Job** | A production work item (braille, large print, tactile graphic, or 3-D print) |
| **Job type** | Category of job: `braille`, `lp_ebraille`, `tactile`, or `print` |
| **Lineage** | The chain of relationships from source files through derivative outputs |
| **METS** | Metadata Encoding and Transmission Standard — an XML framework for describing digital objects |
| **PREMIS** | PREservation Metadata: Implementation Strategies — a standard for digital preservation metadata |
| **Provenance** | The documented history of a file or job, including all transformations and events |
| **SHA-256** | Secure Hash Algorithm 256-bit — used as a fixity check on ingested files |
| **Step** | A workflow milestone within a job (e.g. Digitized, Formatted, Delivered) |
| **Student** | A person record that can be linked to jobs for cross-job production tracking |
| **Tactile graphic** | A raised-line image produced by thermoforming, hand-tooling, or embossing |
| **UEB** | Unified English Braille — the current braille code standard in English-speaking countries |
| **WAL** | Write-Ahead Logging — a SQLite journal mode that improves concurrent access |
